# Фронтенд для системы аутентификации

## Описание

Реализован современный фронтенд для системы аутентификации и авторизации с использованием чистого HTML, CSS и JavaScript.

## Структура

```
auth_system/
├── templates/
│   └── auth/
│       ├── index.html      # Главная страница
│       ├── login.html      # Страница входа
│       ├── register.html   # Страница регистрации
│       └── profile.html     # Страница профиля
├── static/
│   └── auth/
│       ├── css/
│       │   └── styles.css  # Стили приложения
│       └── js/
│           └── api.js      # API утилиты для работы с бэкендом
```

## Страницы

### Главная страница (`/`)
- Редирект на страницу профиля, если пользователь авторизован
- Кнопки для входа и регистрации, если пользователь не авторизован

### Страница входа (`/auth/login/`)
- Форма входа с полями email и пароль
- Валидация полей
- Сохранение токена в localStorage
- Автоматический редирект на профиль после успешного входа
- Ссылка на страницу регистрации

### Страница регистрации (`/auth/register/`)
- Форма регистрации с полями:
  - Email (обязательное)
  - Имя
  - Фамилия
  - Отчество
  - Пароль (обязательное)
  - Подтверждение пароля (обязательное)
- Валидация совпадения паролей
- Обработка ошибок валидации с отображением на соответствующих полях
- Автоматический вход после успешной регистрации
- Ссылка на страницу входа

### Страница профиля (`/auth/profile/`)
- Отображение информации о пользователе:
  - Email
  - Имя, Фамилия, Отчество
  - Дата регистрации
  - Последний вход
- Редактирование профиля (имя, фамилия, отчество)
- Выход из системы
- Удаление аккаунта (мягкое удаление)
- Автоматический редирект на страницу входа, если пользователь не авторизован

## API утилиты (`api.js`)

Класс `AuthAPI` предоставляет методы для работы с API:

- `register(userData)` - регистрация пользователя
- `login(email, password)` - вход в систему
- `logout()` - выход из системы
- `getProfile()` - получение профиля пользователя
- `updateProfile(profileData)` - обновление профиля
- `deleteAccount()` - удаление аккаунта
- `isAuthenticated()` - проверка авторизации
- `getUser()` - получение данных пользователя из localStorage

## Особенности

1. **Управление токенами**: Токены автоматически сохраняются в localStorage при входе и удаляются при выходе
2. **Обработка ошибок**: Улучшенная обработка различных форматов ошибок от API
3. **Валидация форм**: Клиентская валидация с отображением ошибок на полях
4. **Адаптивный дизайн**: Современный UI с поддержкой мобильных устройств
5. **Защита маршрутов**: Автоматические редиректы для неавторизованных пользователей

## Использование

1. Запустите Django сервер:
   ```bash
   python manage.py runserver
   ```

2. Откройте в браузере:
   - Главная страница: `http://localhost:8000/`
   - Вход: `http://localhost:8000/auth/login/`
   - Регистрация: `http://localhost:8000/auth/register/`
   - Профиль: `http://localhost:8000/auth/profile/`
